## Flagship Systems

Four platforms sequenced for delivery — each built with problem framing, architecture, and stack.

### FundsLink Academy — National Education Funding Platform

**Problem:** 342,000+ students excluded yearly due to fragmented bursary applications and unused funding sources scattered across disconnected portals.

**Solution:** A single digital front door for funding, with structured eligibility capture, centralized applications, and a planned AI-assisted matching layer (FastAPI + LangChain).

**Architecture:** Next.js Frontend → Next.js API Routes → Service Layer → Prisma → PostgreSQL
                         ↓
              Python AI Service (FastAPI) — automated scholarship matching

**Status:** Phase 1 — Active Development

### Maphophe Community System — Digital Governance for Villages

**Problem:** Rural villages lack digital infrastructure for governance, communication, and service delivery.

**Solution:** A village-level platform for announcements, service requests, and community voting.

**Architecture:** Next.js Frontend → Next.js API Routes → Service Layer → Prisma → PostgreSQL

**Status:** Phase 2 — Design Complete

### KSDRILL Reserve Bank — Discipline-Driven Savings Banking

**Problem:** Many people struggle with consistent savings habits and lack accessible microfinance tools.

**Solution:** A digital savings platform with goal-setting, automated deposits, and interest tracking.

**Architecture:** Next.js Frontend → Next.js API Routes → Service Layer → Prisma → PostgreSQL

**Status:** Phase 3 — Design Complete

### SyncUp — Creator Collaboration Platform

**Problem:** Creators struggle to find collaborators; existing platforms lack structured negotiation.

**Solution:** A platform with template-based negotiation, 10-message limit, privacy protection, and drop subscriptions.

**Architecture:** Next.js Frontend → Next.js API Routes → Service Layer → Prisma → PostgreSQL

**Status:** Phase 4 — Design Complete
